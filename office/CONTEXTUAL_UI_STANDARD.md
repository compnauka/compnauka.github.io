# CONTEXTUAL_UI_STANDARD.md

Версія: 1.3  
Статус: обов'язковий поведінковий стандарт  
Сфера дії: АРТ Текст, АРТ Таблиці, АРТ Малюнки, АРТ Слайди, АРТ Схеми, АРТ Вектор

---

## 1. Призначення

Цей документ визначає, **де і як повинні з'являтися контекстні інструменти** в АРТ Офіс.

Головна ідея:

- контекст має бути **поруч із дією**;
- контекстні інструменти не повинні без потреби займати постійне місце в shell;
- контекстний UI не повинен дублювати основну панель інструментів;
- для молодших учнів контекстні інструменти мають бути стабільними та передбачуваними.

---

## 2. Базове правило

За замовчуванням **заборонено** реалізовувати контекстні дії як постійний другий ряд під toolbar, якщо цей ряд:

- дублює кнопки з основної панелі;
- займає цінну вертикальну висоту документа або полотна;
- з'являється без чіткого пояснення причин;
- ускладнює інтерфейс для учнів.

Такий ряд допускається лише як **sticky strip** у сервісах для молодших класів або в редакторах, де властивості об'єкта потрібно тримати стабільно видимими.

---

## 3. Дозволені патерни контекстного UI

### 3.1. Inline mini-toolbar

Плаваюча компактна смуга поруч із виділенням або над ним.

Підходить для:
- тексту;
- коротких текстових фрагментів у слайдах;
- швидкого форматування підписів.

Правила:
- містить лише 3–8 найчастіших дій;
- не перекриває курсор і саме виділення;
- зникає, коли контекст зник;
- не використовується для складних властивостей типу таблиці, діаграми або фігури з великою кількістю параметрів.

### 3.2. Context menu

Меню за правим кліком або клавішею контексту.

Підходить для:
- таблиць;
- комірок і діапазонів;
- блоків у схемах;
- об'єктів на полотні;
- швидких команд виду «додати / видалити / дублювати / вирівняти».

Правила:
- дублює локальні дії, а не весь головний toolbar;
- має працювати з клавіатури;
- має закриватися на Escape і поза-клік;
- не повинен містити довгі форми або важкі налаштування.

### 3.3. Inspector

Стабільна панель властивостей у фіксованій зоні або popover-inspector біля об'єкта.

Підходить для:
- фігур;
- зображень;
- діаграм;
- векторних об'єктів;
- складних блоків і конекторів.

Правила:
- inspector використовується для параметрів, які потребують більших контролів;
- він не повинен дублювати меню та toolbar один в один;
- якщо inspector постійний, він не повинен відбирати непропорційно багато місця в workspace.

### 3.4. Sticky strip

Закріплена коротка смуга параметрів під toolbar або у фіксованому місці workspace.

Підходить для:
- молодшої школи;
- Малюнків;
- Схем;
- простих дитячих режимів редагування, де параметри мають бути стабільно видимими.

Правила:
- використовується лише коли mini-toolbar або context menu можуть розгубити учня;
- повинна бути короткою;
- не повинна дублювати всю основну панель;
- має показувати лише властивості поточного типу елемента.

---

## 4. Вибір патерна за типом сервісу

### АРТ Текст
- **text selection** → inline mini-toolbar;
- **table cell / table selection** → context menu або small popover;
- **image** → inspector або popover-inspector.

### АРТ Таблиці
- **cell / range** → context menu або local popover;
- **chart** → inspector.

### АРТ Малюнки
- **brush / shape / text** → sticky strip або simple inspector;
- не використовувати нестабільний «миготливий» contextbar як головний механізм для молодших учнів.

### АРТ Слайди
- **text** → inline mini-toolbar;
- **shape / image** → inspector або popover;
- **slide-level commands** залишаються в toolbar і меню.

### АРТ Схеми
- **block / connector** → sticky strip або inspector;
- швидкі локальні команди можуть бути в context menu.

### АРТ Вектор
- **shape / line / text** → inspector як основний патерн;
- mini-toolbar дозволений лише для коротких швидких дій;
- великий дублюючий другий ряд заборонений.

---

## 5. Антипатерни

Заборонено:

- постійний другий ряд під toolbar, якщо він дублює toolbar;
- великі контекстні панелі, які штовхають документ або полотно вниз;
- використання різних патернів для однакового сценарію без причини;
- приховування важливих дитячих параметрів у нестабільному floating UI без альтернативи;
- мікс з трьох різних механік одночасно в одному сервісі без явної системи.

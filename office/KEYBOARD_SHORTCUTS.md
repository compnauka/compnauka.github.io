# KEYBOARD_SHORTCUTS.md

# Єдиний стандарт клавіатурних скорочень для Офіс ПЛЮС

Версія: 1.0  
Статус: обов'язковий нормативний документ

---

## 1. Призначення

Цей документ фіксує клавіатурний контракт для всієї лінійки Офіс ПЛЮС.  
Однакові дії повинні мати однакові скорочення в усіх редакторах, якщо немає окремо задокументованого винятку.

---

## 2. Загальні правила

1. Глобальні скорочення однакові в усіх сервісах.
2. Локальний сервіс не може перевизначати глобальне скорочення без окремого винятку в документації.
3. Якщо нативний елемент браузера має усталену поведінку, сервіс не повинен її ламати без дуже вагомої причини.
4. Усі скорочення, показані в меню, мають реально працювати.
5. Якщо дія недоступна, скорочення не повинно викликати помилку чи "тиху" поломку.

---

## 3. Глобальні скорочення для всієї лінійки

| Дія | Скорочення | Статус |
|---|---|---|
| Новий | Ctrl+N | обов'язково |
| Відкрити | Ctrl+O | обов'язково |
| Зберегти | Ctrl+S | обов'язково |
| Скасувати | Ctrl+Z | обов'язково |
| Повернути | Ctrl+Y | обов'язково |
| Виділити все | Ctrl+A | обов'язково |
| Копіювати | Ctrl+C | обов'язково |
| Вставити | Ctrl+V | обов'язково |
| Вирізати | Ctrl+X | обов'язково |
| Закрити меню / popup / modal | Esc | обов'язково |

### Додаткові загальні правила
- `Esc` спочатку закриває верхній активний overlay-рівень: dropdown → popover → modal → режим інструмента.
- `Tab` і `Shift+Tab` мають рухати фокус уперед / назад по видимих інтерактивних елементах.
- `Enter` активує меню-елемент, кнопку або вибір у dropdown, якщо фокус стоїть на ньому.

---

## 4. Форматування тексту

Ці скорочення обов'язкові в сервісах, де є rich text або текстове форматування:

| Дія | Скорочення |
|---|---|
| Жирний | Ctrl+B |
| Курсив | Ctrl+I |
| Підкреслення | Ctrl+U |
| Знайти | Ctrl+F |

Застосовується до:
- ПЛЮС Текст
- ПЛЮС Слайди (для текстових об'єктів)
- ПЛЮС Вектор (для текстових об'єктів, якщо підтримується rich formatting)

## 6. Поведінка скорочень у workspace

1. Якщо фокус у `contenteditable`, текстові скорочення мають працювати по виділенню.
2. Якщо фокус у `canvas` або `svg`-workspace, сервісні скорочення мають працювати по виділеному об'єкту або режиму.
3. Якщо відкрита modal, глобальні скорочення документа не повинні "пробивати" модальне вікно, крім дозволених у modal-context.
4. Якщо відкрите menubar/dropdown, `Esc`, стрілки та `Enter` пріоритетно керують меню.

---

## 7. Відображення скорочень

- У меню скорочення вирівнюються по правому краю списку.
- Формат запису всюди однаковий: `Ctrl+S`, `Shift+Tab`, `Ctrl+Shift+G`.
- Не використовувати в одному сервісі `Ctrl + S`, а в іншому `CTRL+S`; запис має бути єдиний.

---

## 8. Заборонено

- перевизначати `Ctrl+S`, `Ctrl+Z`, `Ctrl+Y`, `Ctrl+O`, `Ctrl+N`;
- ламати `Tab` і `Shift+Tab` у формі чи modal без focus trap;
- використовувати `Esc` для нестабільних локальних сценаріїв, якщо відкритий modal чи dropdown;
- показувати скорочення в меню, якщо вони не працюють.

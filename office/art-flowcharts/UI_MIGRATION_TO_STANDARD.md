# UI_MIGRATION_TO_STANDARD.md — ПЛЮС Схеми

## Мета
Привести ПЛЮС Схеми до стандарту лінійки, не втративши сильну спеціалізацію сервісу під блок-схеми.

## Поточні розриви, які треба закрити
1. Стабілізувати дії над вибраним елементом: вони не повинні "мигати" для молодших користувачів.
2. Додати або уніфікувати focus-visible для полотна.
3. Звірити shortcuts із `KEYBOARD_SHORTCUTS.md`.
4. Уніфікувати dropdown/picker і modal-сценарії.
5. Перевірити, що toolbar не дублює бібліотеку блоків.

## Порядок міграції

### Крок 1. Toolbar
- залишити лише керування схемою;
- тримати вставку блоків у лівій бібліотеці.

### Крок 2. Дії над вибраним
- закріпити їх у стабільному місці;
- не ховати основні властивості блока в динамічний contextbar.

### Крок 3. Workspace
- додати focusable container для полотна;
- додати видимий focus-visible;
- забезпечити повернення фокуса після picker/modal.

### Крок 4. Поведінка
- confirm/clear/export dialogs — за `MODAL_STANDARD.md`;
- menus, color pickers, route pickers — за `DROPDOWN_STANDARD.md`.

### Крок 5. QA
- прогнати `COMPONENT_CHECKLIST.md`;
- окремо перевірити зрозумілість для учня початкової і середньої школи.

## Додатково щодо контекстного UI

- властивості блока і зв'язку тримати у sticky strip або inspector;
- не робити нестабільний contextbar як основний спосіб керування;

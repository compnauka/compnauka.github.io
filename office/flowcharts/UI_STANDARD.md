# UI_STANDARD.md — ПЛЮС Схеми

Сервіс: ПЛЮС Схеми  
Тип: редактор блок-схем  
Основна аудиторія: 3–8 класи

## 1. Меню
`Файл · Редагування · Вставка · Перегляд · Допомога`

## 2. Ліва бібліотека блоків
Залишається постійною і є основним місцем створення блоків:
- Початок / Кінець
- Дія
- Умова
- Ввід / Вивід
- Підпрограма
- З'єднувач

## 3. Панель інструментів
Групи:
- Файл
- Історія
- Індикатор вибраного елемента
- Дії над вибраним
- Прив'язка
- Масштаб

## 4. Контекст / inspector
Для молодших користувачів властивості вибраного блока або зв'язку повинні бути стабільно доступні:
- sticky context panel;
- inspector-панель;
- або чітко закріплена група дій на toolbar.

### 4.1. Що показувати
- текст;
- маршрут;
- підпис;
- колір;
- видалення.

## 5. Робоча область
- ліва бібліотека блоків;
- центральне полотно;
- без дублювання вставки блоків на toolbar;
- полотно повинно мати focus-visible.

## 6. Статус-бар
- вибраний блок або зв'язок;
- кількість блоків;
- прив'язка;
- масштаб.

## 7. Гарячі клавіші
Відповідно до `KEYBOARD_SHORTCUTS.md`:
- Delete — видалити вибране
- Ctrl+D — дублювати
- G — переключити прив'язку
- Esc — закрити overlay або скасувати режим

## 8. Заборонено
- дублювати бібліотеку блоків у верхньому toolbar;
- ховати ключові дії над блоком у нестабільний contextbar;
- робити непомітний selection state або непомітний focus state полотна.

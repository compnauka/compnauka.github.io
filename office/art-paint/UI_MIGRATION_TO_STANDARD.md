# UI_MIGRATION_TO_STANDARD.md — АРТ Малюнки

## Мета
Привести АРТ Малюнки до єдиного стандарту АРТ Офіс без збільшення когнітивного навантаження.

## Поточні розриви, які треба закрити
1. Меню потрібно узгодити з лінійкою: додати `Вставка`, лишивши `Інструменти`.
2. Розвести логіку:
   - вставити об'єкт;
   - обрати інструмент.
3. Для молодшої аудиторії уникнути contextbar, який з'являється і зникає без пояснення.
4. Додати focus-visible для полотна або його контейнера.
5. Звірити shortcuts із `KEYBOARD_SHORTCUTS.md`.
6. Палітри й picker мають працювати за `DROPDOWN_STANDARD.md`.
7. Діалоги очищення полотна / параметрів мають відповідати `MODAL_STANDARD.md`.

## Порядок міграції

### Крок 1. Меню
- зробити каркас `Файл · Редагування · Вставка · Інструменти · Перегляд · Допомога`;
- перенести штампи / текст / вставку зображення у логіку `Вставка`.

### Крок 2. Toolbar
- залишити зліва: Новий, Відкрити, Зберегти, Скасувати, Повернути;
- згрупувати інструменти;
- швидкі об'єкти тримати окремо від інструментів.

### Крок 3. Контекстні властивості
- або зробити sticky panel;
- або перенести базові параметри інструмента на основну панель;
- не залишати критичні властивості лише у нестабільному overlay.

### Крок 4. Workspace
- додати `tabindex="0"` для контейнера полотна, якщо потрібно;
- додати видимий focus-visible;
- після закриття picker фокус має повертатися на trigger або на полотно.

### Крок 5. Поведінка overlays і modal
- shape-picker / stamp-picker / color-picker — за `DROPDOWN_STANDARD.md`;
- confirm-дії — за `MODAL_STANDARD.md`.

### Крок 6. QA
- прогнати `COMPONENT_CHECKLIST.md`;
- окремо перевірити дитячу зрозумілість для 1–5 класу.

## Додатково щодо контекстного UI

- не використовувати миготливий плаваючий contextbar як головний патерн;
- параметри інструмента тримати в sticky strip або простому inspector;

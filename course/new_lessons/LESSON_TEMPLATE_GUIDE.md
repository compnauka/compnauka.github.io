# Універсальний шаблон уроку

Цей проєкт працює на основі об'єкта шаблону уроку в `js/lesson-data.js`.
Підтримуються два режими:
- `student` — видно тільки те, що потрібно дитині для проходження уроку.
- `teacher` — видно повний зміст, методичні пояснення, поради та підготовчі блоки.

Важливе правило для шкільного використання:
- стан уроку не зберігається між відкриттями сторінки;
- після оновлення сторінки або повторного відкриття браузера урок стартує з початку;
- це потрібно для спільних комп'ютерів у школі, щоб наступна дитина не бачила чужий прогрес.

## Як змінювати тему

1. Створіть файл у `js/lessons/`, наприклад `my-topic-1-2.js`.
2. Експортуйте об'єкт шаблону уроку.
3. Додайте тему в `js/lessons/catalog.js`.
4. Перемикайте тему через список у шапці або через URL-параметр `?lesson=...`.
5. Збережіть структуру ключів, щоб UI автоматично зібрав:
- назву, мету, цілі
- вступний блок підготовки або плану (`overview`)
- основну частину (`sections`)
- практичні завдання (`activities`)
- підсумковий тест (`quiz`)
- рефлексію (`reflection`)

Приклад URL:
- `index.html?lesson=info-types-1-2`
- `index.html?lesson=message-actions-1-2`

Окремі сторінки для поширення та SEO:
- `info-types-1-2.html`
- `message-actions-1-2.html`
- `info-history-coding-1-2.html`

## Режими

- `student` показує тільки навчальний контент для дитини.
- `teacher` показує повний контент + методичні підказки (`teacherTip`, `goalNote`, `reflection.note`).

## Мінімальна структура

```js
{
  title: "Тема уроку",
  studentHook: "Короткий мотиваційний гачок для дитини",
  teacherOverview: "Короткий опис для вчителя",
  studentMeta: ["лише те, що доречно учню"],
  teacherMeta: ["клас", "тривалість", "формат", "особливості уроку"],
  overview: {
    label: "Підготовка",
    title: "Що варто передбачити",
    cards: [
      { title: "Ключові слова", items: ["..."] },
      { title: "Матеріали", audience: "teacher", items: ["..."] },
      { title: "Для учня", audience: "student", text: "..." }
    ]
  },
  goal: "Мета",
  goalNote: "Порада вчителю",
  objectives: ["..."],
  sections: [
    {
      title: "...",
      intro: "...",
      audience: "both", // both | student | teacher
      cards: [{ title: "...", text: "..." }],
      teacherTip: "..."
    }
  ],
  activities: {
    draw: { id: "draw", type: "draw", ... },
    classify: { id: "classify", type: "classify", ... },
    truefalse: { id: "truefalse", type: "truefalse", ... },
    pick: { id: "pick", type: "pick", ... },
    fill: { id: "fill", type: "fill", ... },
    scenarios: { id: "scenarios", type: "scenarios", ... },
    creative: { id: "creative", type: "creative", ... }, // опційно
    transfer: { id: "transfer", type: "transfer", ... } // опційно
  },
  quiz: { count: 4, questions: [...] },
  reflection: { prompt: "...", options: [...], note: "..." }
}
```

## Що робить шаблон універсальним

- `overview` — необов'язковий блок для матеріалів, ключових слів, сценарію уроку, домашнього продовження, підготовки вчителя.
- `audience` у `overview.cards[]` та `sections[]` — дає змогу показувати блоки лише учню, лише вчителю або обом.
- Стан уроку не зберігається між сесіями: після оновлення сторінки все повертається до початкового стану.
- Предметний зміст живе в `js/lessons/*.js`, а логіка інтерфейсу не залежить від теми.
- `creative` дає коротке творче завдання без довгого друкування: учень збирає власний приклад із готових карток.
- `transfer` перевіряє перенесення знання в нову життєву ситуацію, а не лише впізнавання знайомого прикладу.

## Правило першого блоку

- У режимі учня перший екран має містити мотиваційний гачок: коротку фразу, яка викликає цікавість до теми.
- У режимі учня не слід показувати службову інформацію на кшталт тривалості уроку, формату проведення чи методичних особливостей.
- У режимі вчителя перший екран може містити тривалість, формат, організаційні деталі, коротку методичну рамку.
- Якщо тема має окремі `studentMeta` і `teacherMeta`, для учня залишайте тільки те, що допомагає увійти в тему, а не адмініструвати урок.

## Важливо для доступності

- Тексти кнопок і заголовків мають бути короткими та однозначними.
- Для кожного нового інтерактиву лишайте текстовий фідбек (`task-feedback`).
- Не видаляйте `aria-live`, `skip-link`, `focus-visible` стилі.

# Універсальний шаблон уроку

Проєкт побудований на шаблонах уроків у `js/lessons/*.js` і підтримує два режими:
- `student` — видно лише те, що потрібно дитині.
- `teacher` — видно повний зміст, підготовку, методичні поради та пояснення.

Важливе правило для шкільного використання:
- стан уроку не зберігається між відкриттями сторінки;
- після оновлення сторінки або повторного відкриття браузера урок стартує з початку;
- це потрібно для спільних комп'ютерів у школі, щоб наступна дитина не бачила чужий прогрес.

## Рекомендована 6-урочна лінійка

1. `Інформація. Як ми сприймаємо інформацію`
2. `Види інформації та способи подання`
3. `Дії з інформацією. Інформаційний процес`
4. `Об’єкти, властивості, моделі`
5. `Кодування інформації`
6. `Джерела інформації. Правдиве і неправдиве`

Саме ця структура краще узгоджується з конкретними результатами:
- [2 ІФО 1.1]
- [2 ІФО 1.2.1]
- [2 ІФО 1.3.1]
- [2 ІФО 1.4.1]

## Наступний цикл

Після цього циклу планується новий тематичний блок про комп’ютери:
- що таке комп’ютер;
- види комп’ютерів;
- основні складові;
- просте призначення пристроїв.

Це потрібно враховувати в індексі, навігації та подальшому плануванні контенту.

## Правило першого блоку

- У режимі учня перший екран має містити мотиваційний гачок, а не службову картку уроку.
- Учню не слід показувати тривалість, формат уроку або методичні особливості.
- У режимі вчителя перший екран може містити тривалість, формат, результат навчання та коротку методичну рамку.
- Для цього використовуйте окремі поля `studentHook`, `studentMeta`, `teacherOverview`, `teacherMeta`.

## Мінімальна структура шаблону

```js
{
  id: "lesson-slug",
  title: "Назва уроку",
  studentHook: "Короткий мотиваційний гачок для дитини",
  teacherOverview: "Коротка рамка для вчителя",
  studentMeta: ["..."],
  teacherMeta: ["..."],
  overview: { ... },
  goal: "...",
  goalNote: "...",
  objectives: ["..."],
  sections: [ ... ],
  activities: {
    draw: { ... },
    classify: { ... },
    sequence: { ... },
    truefalse: { ... },
    pick: { ... },
    fill: { ... },
    scenarios: { ... },
    creative: { ... },
    transfer: { ... }
  },
  quiz: { count: 4, questions: [...] },
  reflection: { prompt: "...", options: [...], note: "..." }
}
```

## Що робить шаблон універсальним

- `overview` дозволяє мати окремі картки для учня й учителя.
- `audience` у `overview.cards[]` і `sections[]` дозволяє показувати блоки лише потрібній аудиторії.
- `creative` дає коротке творче завдання без довгого друкування.
- `transfer` перевіряє перенесення знання в нову життєву ситуацію.
- Предметний зміст живе в `js/lessons/*.js`, а логіка інтерфейсу не залежить від теми.

## Педагогічні правила

- Один урок має тримати 1 головну ідею і 2-3 ключові результати.
- У базовому маршруті не перевантажуйте дітей абстрактними термінами.
- Після вправ бажано ставити коротке питання «чому саме так?».
- У кожному уроці бажано мати хоча б одну вправу на перенесення знання в життєву ситуацію.
- У формулюваннях для 1-2 класу пріоритет мають короткі, конкретні й віково зрозумілі фрази.

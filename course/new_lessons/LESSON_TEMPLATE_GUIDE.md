# Універсальний шаблон уроку

Проєкт побудований на шаблонах уроків у `js/lessons/*.js` і підтримує два режими:
- `student` — видно лише те, що потрібно дитині.
- `teacher` — видно повний зміст, підготовку, методичні поради та пояснення.

Важливе правило для шкільного використання:
- стан уроку не зберігається між відкриттями сторінки;
- після оновлення сторінки або повторного відкриття браузера урок стартує з початку;
- це потрібно для спільних комп'ютерів у школі, щоб наступна дитина не бачила чужий прогрес.

## Рекомендована 8-урочна лінійка

1. `Інформація. Як ми сприймаємо інформацію`
2. `Види інформації та способи подання`
3. `Дії з інформацією. Інформаційний процес`
4. `Об’єкти, властивості, моделі`
5. `Кодування інформації`
6. `Джерела інформації. Правдиве і неправдиве`
7. `Множини. Групуємо та впорядковуємо`
8. `Прості схеми та таблиці`

Саме ця структура краще узгоджується з конкретними результатами:
- [2 ІФО 1.1]
- [2 ІФО 1.2.1]
- [2 ІФО 1.3.1]
- [2 ІФО 1.4.1]

Важлива примітка для планування:
- цей цикл не слід описувати як повне покриття всього стандарту 1-2 класу;
- зараз це саме `Цикл 1. Інформація та робота з нею`;
- у landing та вчительському режимі варто явно показувати, які результати вже покрито повністю, а які лише частково.

## Наступний цикл

Після цього циклу планується новий тематичний блок про комп’ютери:
- що таке комп’ютер;
- види комп’ютерів;
- основні складові;
- просте призначення пристроїв.

Це потрібно враховувати в індексі, навігації та подальшому плануванні контенту.

Після блоку про комп’ютери курс має продовжуватися темами 1-2 класу на тій самій платформі:
- програми та вікна програм;
- алгоритми і виконавці;
- безпечна й охайна робота з комп’ютером;
- практичні вправи на використання програм у навчальних ситуаціях.

Агент, який продовжує курс, не повинен вигадувати нову структуру з нуля. Потрібно:
- зберігати єдиний підхід до шаблону;
- додавати нові тематичні цикли послідовно;
- підтримувати спільну навігацію індексу як «інтерактивного підручника», а не набору розрізнених сторінок.

## Дорожня карта тем 1-2 класу

Поточний цикл уже реалізовано:
1. `Інформація. Як ми сприймаємо інформацію`
2. `Види інформації та способи подання`
3. `Дії з інформацією. Що ми робимо з повідомленнями`
4. `Об’єкти, властивості, моделі`
5. `Кодування інформації`
6. `Джерела інформації. Правдиве і неправдиве`
7. `Множини. Групуємо та впорядковуємо`
8. `Прості схеми та таблиці`

Рекомендований наступний цикл `Комп’ютери`:
1. `Що таке комп’ютер і де ми його зустрічаємо`
2. `Які бувають комп’ютери`
3. `Основні частини комп’ютера`
4. `Для чого потрібні різні пристрої`
5. `Коли щось не виходить: помічаю проблему і звертаюся по допомогу`
6. `Як працювати безпечно і дбайливо`

Після нього рекомендований цикл `Програми`:
1. `Що таке програма`
2. `Які програми допомагають вчитися і творити`
3. `Вікно програми і прості дії в ньому`
4. `Відкрити, виконати дію, закрити`

Після нього рекомендований цикл `Алгоритми`:
1. `Команди і виконавці`
2. `Послідовність дій`
3. `Алгоритм у побутовій ситуації`
4. `Пошук помилки в простому алгоритмі`

Якщо агент додає новий цикл, починати треба саме з цієї дорожньої карти, а не з випадкового вибору тем.

## Карта покриття результатів

Поточний цикл `Інформація та робота з нею` працює насамперед на:
- [2 ІФО 1.1] — поки що частково;
- [2 ІФО 1.2] — поки що частково;
- [2 ІФО 1.3] — поки що частково;
- [2 ІФО 1.4] — покрито в межах циклу.

Наступні цикли мають будуватися від мапи результатів, а не лише від переліку зручних тем:
- `Цикл 2. Комп’ютери та цифровий світ` → [2 ІФО 3.1], [2 ІФО 3.2], [2 ІФО 4.1]
- `Цикл 3. Алгоритми довкола нас` → [2 ІФО 2.1], [2 ІФО 2.2], [2 ІФО 2.3]
- `Цикл 4. Творимо та ділимося` → [2 ІФО 2.4], [2 ІФО 2.5], [2 ІФО 4.3]
- `Цикл 5. Безпека та спілкування онлайн` → [2 ІФО 3.3], [2 ІФО 4.2], частина [2 ІФО 1.4]

## Правило першого блоку

- У режимі учня перший екран має містити мотиваційний гачок, а не службову картку уроку.
- Учню не слід показувати тривалість, формат уроку або методичні особливості.
- У режимі вчителя перший екран може містити тривалість, формат, результат навчання та коротку методичну рамку.
- Для цього використовуйте окремі поля `studentHook`, `studentMeta`, `teacherOverview`, `teacherMeta`.

## Мінімальна структура шаблону

```js
{
  id: "lesson-slug",
  title: "Назва уроку",
  studentHook: "Короткий мотиваційний гачок для дитини",
  teacherOverview: "Коротка рамка для вчителя",
  studentMeta: ["..."],
  teacherMeta: ["..."],
  coverage: {
    cycle: "Цикл 1. Інформація та робота з нею",
    note: "Що саме покриває цей урок",
    results: [{ code: "2 ІФО 1.1", status: "partial", focus: "...", next: "..." }]
  },
  overview: { ... },
  goal: "...",
  goalNote: "...",
  objectives: ["..."],
  sections: [ ... ],
  activities: {
    draw: { ... },
    classify: { ... },
    sequence: { ... },
    truefalse: { ... },
    pick: { ... },
    fill: { ... },
    "table-read": { ... },
    scenarios: { ... },
    creative: { ... },
    transfer: { ... }
  },
  quiz: { count: 4, questions: [...] },
  reflection: { prompt: "...", options: [...], note: "..." }
}
```

## Що робить шаблон універсальним

- `overview` дозволяє мати окремі картки для учня й учителя.
- `audience` у `overview.cards[]` і `sections[]` дозволяє показувати блоки лише потрібній аудиторії.
- `coverage` дає вчителю коротке пояснення, який саме результат покриває урок і що ще залишається на наступні заняття.
- `creative` дає коротке творче завдання без довгого друкування.
- `transfer` перевіряє перенесення знання в нову життєву ситуацію.
- `table-read` дає просту таблицю, де дитина знаходить потрібну клітинку за рядком і стовпцем.
- Предметний зміст живе в `js/lessons/*.js`, а логіка інтерфейсу не залежить від теми.

## Педагогічні правила

- Один урок має тримати 1 головну ідею і 2-3 ключові результати.
- У базовому маршруті не перевантажуйте дітей абстрактними термінами.
- Після вправ бажано ставити коротке питання «чому саме так?».
- У кожному уроці бажано мати хоча б одну вправу на перенесення знання в життєву ситуацію.
- У формулюваннях для 1-2 класу пріоритет мають короткі, конкретні й віково зрозумілі фрази.
- Формулювання мають бути не просто короткими, а доступними: через знайомі дитині приклади (`дзвоник`, `смайлик`, `розклад`, `мишка`, `кнопка`, `малюнок`, `вікно програми`).
- Блок `overview.title` зараз використовує живе формулювання `Що підготувати до уроку`. Не повертайтеся до штучних формулювань на кшталт `Що варто передбачити до уроку`.
- Якщо урок містить вправу на порядок дій, для дітей 1-2 класу краще використовувати дві зони: список кроків і окремі пронумеровані місця для заповнення.
- Якщо урок заявляє покриття результату стандарту, у полі `coverage` треба чесно позначати `full` або `partial`, а не перебільшувати.

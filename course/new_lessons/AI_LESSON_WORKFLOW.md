# AI Lesson Workflow (Internal Playbook)

Це робоча інструкція для швидкого додавання нових уроків у поточну систему.
Мета: користувач дає тему й зміст, далі агент самостійно збирає урок, підключає його до каталогу, генерує html-сторінку й перевіряє результат.

Ключове правило після розширення курсу:
- нові уроки й цикли плануються не лише від теми, а від карти очікуваних результатів [2 ІФО ...].

## 1) Як влаштована система

- `index.html` — головна сторінка підручника: перемикач **Учень / Вчитель** (як на сторінці уроку), контейнери `#landing-student-modules` та `#landing-teacher-modules`.
- `js/landing.js` — будує вміст модулів: для учня — прості описи та список уроків із посиланнями; для вчителя — зведення кодів ІФО по модулю та розгорнуті картки уроків (мета, `coverage`, `objectives` з шаблонів у `js/lessons/*.js`).
- `js/landing-modules.js` — карта **п’яти модулів** курсу (`textbookModules`): назва, короткі описи для учня/вчителя, упорядкований масив `lessonIds` (має збігатися з порядком у `js/lessons/catalog.js` у межах кожного блоку).
- `js/lessons/*.js` — контентні файли уроків.
- `js/lessons/catalog.js` — реєстр уроків.
- `lesson-page.template.html` — шаблон SEO-сторінок уроків.
- `scripts/generate-lesson-pages.ps1` — генератор окремих html-сторінок.
- `js/app.js` — ядро рендерингу уроку.
- `js/lesson-data.js` — фабрика уроку, яка будує секції, активності та тест.
- `js/task-*.js` — модулі інтерактивних завдань.

### 1.1) Імена HTML-сторінок уроків

- Файли: `m{модуль 1–5}-{номер у модулі двома цифрами}-{slug}.html` (наприклад `m2-03-computer-parts.html`).
- Стабільний **технічний id** уроку (`id` у каталозі, `data-lesson-id` на `<body>`, ім’я файлу шаблону `js/lessons/<id>.js`) залишається з суфіксом `-1-2`, щоб не перейменовувати всю логіку застосунку.
- Після змін у каталозі треба оновити `url` у `catalog.js`, запис `File` у `scripts/generate-lesson-pages.ps1`, перегенерувати сторінки та оновити шляхи в тестах, якщо змінювалися імена файлів.

## 2) Базові правила для шкільного сценарію

- урок не повинен зберігати прогрес між відкриттями сторінки;
- після refresh або повторного відкриття браузера все має починатися з нуля;
- це обов’язкова умова для спільних шкільних комп’ютерів;
- перший блок у режимі учня має бути мотиваційним гачком, а не службовою карткою уроку;
- у режимі вчителя перший блок може містити тривалість, формат, організаційні деталі та коротку методичну рамку.
- заголовок підготовчого блоку зараз зафіксовано як `Що підготувати до уроку`;
- порожні плашки зворотного зв’язку не повинні бути видимими;
- вправи на впорядкування для 1-2 класу треба подавати через дві зони: набір кроків і пронумеровані місця для складання порядку.

## 3) Рекомендована навчальна структура

Основна лінійка для 1-2 класу:
1. `Інформація. Як ми сприймаємо інформацію`
2. `Види інформації та способи подання`
3. `Дії з інформацією. Що ми робимо з повідомленнями`
4. `Об’єкти, властивості, моделі`
5. `Кодування інформації`
6. `Джерела інформації. Правдиве і неправдиве`
7. `Множини. Групуємо та впорядковуємо`
8. `Прості схеми та таблиці`

Ця структура краще узгоджується з результатами:
- [2 ІФО 1.1]
- [2 ІФО 1.2.1]
- [2 ІФО 1.3.1]
- [2 ІФО 1.4.1]

Наступні цикли після цієї лінійки:
- `Комп’ютери та цифровий світ`;
- `Алгоритми довкола нас`;
- `Творимо та ділимося`;
- `Безпека та спілкування онлайн`.

Рекомендований порядок циклів для продовження підручника:
1. `Комп’ютери та цифровий світ`
2. `Алгоритми довкола нас`
3. `Творимо та ділимося`
4. `Безпека та спілкування онлайн`

Агент не має змішувати ці цикли в один великий набір уроків. Краще тримати короткі послідовні лінійки по 4-8 уроків.

## 3.2) Планувати від карти результатів

Перед початком нового циклу агент має коротко відповісти на три запитання:
1. Які саме коди [2 ІФО ...] покриває цей цикл?
2. Що вже покрито в попередніх циклах повністю, а що лише частково?
3. Який урок у циклі закриває основну частину результату, а який лише підводить до нього?

Поточна дорожня карта покриття така:
- `Цикл 1. Інформація та робота з нею` → [2 ІФО 1.1], [2 ІФО 1.2], [2 ІФО 1.3], [2 ІФО 1.4]
- `Цикл 2. Комп’ютери та цифровий світ` → [2 ІФО 3.1], [2 ІФО 3.2], [2 ІФО 4.1]
- `Цикл 3. Алгоритми довкола нас` → [2 ІФО 2.1], [2 ІФО 2.2], [2 ІФО 2.3]
- `Цикл 4. Творимо та ділимося` → [2 ІФО 2.4], [2 ІФО 2.5], [2 ІФО 4.3]
- `Цикл 5. Безпека та спілкування онлайн` → [2 ІФО 3.3], [2 ІФО 4.2], частина [2 ІФО 1.4]

## 3.1) З чого починати новий цикл

Якщо треба додати новий цикл, наприклад `Творимо та ділимося`, починати треба в такому порядку:
1. Переглянути вже існуючі уроки в `js/lessons/*.js`.
2. Визначити повний міні-цикл тем, а не один випадковий урок.
3. Для кожного уроку коротко зафіксувати:
   - головну ідею;
   - 2-3 очікувані результати;
   - 3-5 життєвих прикладів для дітей;
   - набір активностей, які найкраще підходять саме до цієї теми.
4. Лише після цього створювати новий `lesson template`.

Стартовий шаблон планування для циклу `Комп’ютери`:
1. `Що таке комп’ютер і де ми його зустрічаємо`
2. `Які бувають комп’ютери`
3. `Основні частини комп’ютера`
4. `Для чого потрібні різні пристрої`
5. `Коли щось не виходить: помічаю проблему і звертаюся по допомогу`
6. `Як працювати безпечно і дбайливо`

Стартовий шаблон планування для циклу `Програми`:
1. `Що таке програма`
2. `Програми для малювання, читання, навчання і гри`
3. `Вікно програми`
4. `Прості дії у програмі`

Стартовий шаблон планування для циклу `Алгоритми`:
1. `Команди і виконавці`
2. `Послідовність дій`
3. `Алгоритм у повсякденній справі`
4. `Знайди і виправ помилку в порядку дій`
5. `Як записати алгоритм`

Стартовий шаблон планування для циклу `Творимо та ділимося`:
1. `Малюємо у програмі`
2. `Створюємо простий інформаційний продукт`
3. `Підписуємо свою роботу і поважаємо авторство`
4. `Працюємо самостійно і разом`

Стартовий шаблон планування для циклу `Безпека та спілкування онлайн`:
1. `Для чого нам інтернет`
2. `Шукаємо інформацію онлайн`
3. `Що можна і що не можна повідомляти`
4. `Спілкуємося чемно онлайн`
5. `Перевіряємо повідомлення перед тим, як вірити`

## 4) Стандартний сценарій додавання нового уроку

1. Створити контентний файл у `js/lessons/<slug>.js`.
2. Експортувати `...LessonTemplate`.
3. Додати урок у `js/lessons/catalog.js`.
4. Оновити `scripts/generate-lesson-pages.ps1`.
5. Запустити:
   - `powershell -ExecutionPolicy Bypass -File scripts\generate-lesson-pages.ps1`
6. Запустити:
   - `powershell -ExecutionPolicy Bypass -File tests\run-tests.ps1`
7. Якщо змінювали рендер, режими або нові активності, запустити:
   - `powershell -ExecutionPolicy Bypass -File tests\run-browser-smoke.ps1`

Якщо додається цілий цикл, а не один урок:
1. Спочатку оновити `js/lessons/catalog.js` у правильному навчальному порядку.
2. Потім оновити `scripts/generate-lesson-pages.ps1` для всіх нових сторінок.
3. Оновити `js/landing-modules.js`: додати новий модуль або розширити масив `lessonIds` існуючого, щоб головна сторінка показувала актуальну структуру (учневі — описи модулів і списки уроків; вчителю — ІФО та картки уроків підтягуються з шаблонів автоматично).
4. Переконатися, що `index.html` містить контейнери для режимів учень/вчитель (див. поточну розмітку), якщо змінювалися лише дані — статичний каркас зазвичай не чіпають.
5. Після генерації сторінок перевірити відображення в обох режимах головної: для учня — без зайвих методичних деталей; для вчителя — коректні коди та формулювання з полів `coverage` і `objectives`.

## 6) Педагогічний чекліст

- Урок має працювати на 1-2 чіткі результати, а не на все одразу.
- Якщо результат покривається не повністю, це потрібно прямо вказати в `coverage`, а не маскувати загальною фразою в `teacherMeta`.
- Для 1-2 класу важливі короткі й конкретні формулювання.
- Пояснення мають іти від життєвого прикладу до узагальнення.
- У кожному уроці бажано мати:
  - одну вправу на впізнавання або класифікацію;
  - одну вправу на пояснення або коротке конструювання відповіді (`creative`);
  - одну вправу на перенесення знання в нову ситуацію (`transfer`).
- Формувальне оцінювання має бути не лише в тесті, а й у `teacherTip`.
- Не використовуйте двозначні приклади там, де дитина може мати кілька однаково логічних відповідей, якщо це не проговорено вчителем.
- Кожен новий цикл має бути педагогічно нарощуваним: від знайомого і конкретного до трохи більш узагальненого.
- Для тем `Комп’ютери`, `Програми`, `Алгоритми` потрібно уникати дорослої технічної лексики без побутової опори.
- Якщо можна сказати простіше через приклад, треба говорити простіше через приклад.

## 7) Технічні правила

- Не додавайте темо-специфічну логіку в `js/app.js` або `js/task-*.js`.
- Усе предметне — лише в `js/lessons/*.js`.
- Не вмикайте постійне збереження прогресу в `localStorage` або іншому сховищі.
- Головна сторінка має називатися `Інтерактивний підручник з інформатики для 1 та 2 класу`. У режимі **Учень** не варто перевантажувати текстом про ІФО чи SEO; у режимі **Вчитель** показується зведення очікуваних результатів по модулях і деталізація по кожному уроку (джерело — `coverage` та `objectives` у шаблонах).
- Окремий блок «дорожня карта наступних циклів» зі старої головної прибрано: структура курсу задається модулями в `js/landing-modules.js` і порядком у `catalog.js`.
- Для нових типів активностей:
  1. створити `js/task-<type>.js`
  2. підключити в `js/activity-registry.js`
  3. додати побудову в `js/lesson-data.js`

Практичне правило для нового агента:
- спочатку використовуй уже наявні типи активностей;
- новий тип активності додавай лише тоді, коли тему справді неможливо добре подати через існуючі `draw`, `classify`, `sequence`, `truefalse`, `pick`, `fill`, `table-read`, `scenarios`, `creative`, `transfer`;
- для читання простих таблиць уже є тип `table-read`, тож для тем про рядки, стовпці й клітинки спершу використовуй його;
- перед зміною логіки існуючих активностей перевір, чи це не погіршить уже створені уроки.

## 8) Definition of Done

- є `js/lessons/<slug>.js`
- є сторінка уроку за схемою `m{модуль}-{NN}-{slug}.html` (див. `js/lessons/catalog.js` і `scripts/generate-lesson-pages.ps1`)
- є запис у `js/lessons/catalog.js`
- урок відкривається з каталогу і напряму по URL
- тест `tests/run-tests.ps1` проходить успішно
- browser smoke `tests/run-browser-smoke.ps1` проходить успішно, якщо змінювали рендер або поведінку в браузері
- після оновлення сторінки урок стартує з чистого стану
- перший блок відрізняється для учня і для вчителя
- контент відповідає віку 1-2 класу і конкретним результатам

## 9) Перші дії нового агента

Якщо агент відкрив цей проєкт уперше, він має починати не з редагування файлів навмання, а з такого короткого маршруту:
1. Прочитати цей файл цілком.
2. Прочитати `LESSON_TEMPLATE_GUIDE.md`.
3. Переглянути 2-3 уже готові lesson templates у `js/lessons/`.
4. Перевірити `js/lessons/catalog.js`, щоб зрозуміти поточний порядок курсу.
5. Переглянути `js/landing-modules.js` і `index.html`, щоб зрозуміти, як головна збирає модулі для учня та вчителя.
6. Лише після цього планувати новий цикл або новий урок.

Якщо завдання звучить як `додай теми про комп’ютери`, агент має:
1. Сформувати цілий міні-цикл на 4-5 уроків.
2. Узгодити його з уже реалізованим інформаційним циклом.
3. Додати уроки в каталог у правильній послідовності.
4. Згенерувати окремі html-сторінки.
5. Перевірити тести.

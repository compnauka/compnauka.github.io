# AI Lesson Workflow (Internal Playbook)

Це робоча інструкція для швидкого додавання нових уроків у поточну систему.
Мета: користувач дає тему + зміст, далі агент самостійно робить усе технічно.

## 1) Як влаштована система

- `index.html` — зонтична сторінка (каталог тем).
- `js/landing.js` — рендер карток уроків на `index.html`.
- `js/lessons/*.js` — контентні файли уроків (дані теми).
- `js/lessons/catalog.js` — реєстр уроків (id, label, url, template).
- `*.html` (окремо на кожен урок) — SEO/URL сторінки з `data-lesson-id`.
- `js/app.js` — ядро рендерингу уроку.
- `js/lesson-data.js` — універсальна фабрика уроку (рандомізація, копії даних).
- `js/task-*.js` — модулі інтерактивних завдань.

Правило для шкільних комп'ютерів:
- урок не повинен зберігати прогрес, вибори чи результати між відкриттями сторінки;
- після refresh, повторного запуску браузера або відкриття уроку іншою дитиною все має починатися з нуля.

## 2) Стандартний сценарій додавання нового уроку

1. Створити контентний файл:
- шлях: `js/lessons/<slug>.js`
- формат: `export const <name>LessonTemplate = { ... }`

2. Додати урок у каталог:
- файл: `js/lessons/catalog.js`
- додати об'єкт:
  - `id`: унікальний slug
  - `label`: назва для селекта/каталогу
  - `url`: `./<slug>.html`
  - `template`: імпортований template

3. Створити окрему HTML-сторінку уроку:
- оновити масив у `scripts/generate-lesson-pages.ps1`
- запустити:
  - `powershell -ExecutionPolicy Bypass -File scripts\generate-lesson-pages.ps1`
- генератор оновить окремі сторінки уроків (`*.html`) із шаблону `lesson-page.template.html`

4. Перевірити, що бейдж зверху клікабельний:
- "Інформатика 1-2 клас" має вести на `./index.html`.

5. Запустити тести:
- `powershell -ExecutionPolicy Bypass -File tests\run-tests.ps1`

## 3) Структура lesson template (обов'язкові поля)

```js
{
  id: "lesson-slug",
  title: "Назва",
  studentHook: "...", // мотиваційний гачок для дитини
  teacherOverview: "...", // коротка рамка для вчителя
  studentMeta: ["..."], // без тривалості та службових деталей
  teacherMeta: ["..."], // можна вказати клас, тривалість, формат
  goal: "...",
  goalNote: "...",
  objectives: ["..."],
  sections: [
    {
      title: "...",
      intro: "...",
      cards: [{ title: "...", text: "..." }], // або bullets: ["..."]
      teacherTip: "..."
    }
  ],
  activities: {
    draw: {...},
    classify: {...},
    sequence: {...}, // опційно
    truefalse: {...},
    pick: {...},
    fill: {...},
    scenarios: {...},
    creative: {...}, // опційно
    transfer: {...} // опційно
  },
  quiz: {
    count: 4,
    questions: [...]
  },
  reflection: {
    prompt: "...",
    options: [...],
    note: "..."
  }
}
```

## 4) Налаштування завдань (шпаргалка)

### `draw`
- Вільне малювання + fallback select.
- Ключі: `fallbackOptions`, `teacherTip`.

### `classify`
- Розкладання карток по категоріях.
- Ключі: `categories`, `categoryIcons`, `perCategory`, `items[{label, emoji, correct}]`.

### `sequence` (опційно)
- Порядок кроків (історія, ланцюжок повідомлення, алгоритм).
- Ключі: `steps` (масив рядків у правильному порядку).

### `truefalse`
- Твердження правда/хиба.
- Ключі: `count`, `statements[{text, answer:boolean, emoji}]`.

### `pick`
- "Знайди зайве".
- Ключі: `count`, `groups[{question, options[{label, correct}]}]`.

### `fill`
- Вибір слова у пропуск.
- Ключі: `count`, `sentences[{text, answer, options[]}]`.

### `scenarios`
- Ситуації з кількома правильними відповідями.
- Ключі: `count`, `situations[{emoji, text, options[{label, correct}]}]`.

### `creative` (опційно)
- Коротке творче завдання без довгого друкування.
- Учень збирає власний приклад із готових карток.
- Ключі: `groups[{title, options[{label, value?}]}]`, `previewTemplate`, `placeholder`.

### `transfer` (опційно)
- Завдання на перенесення знання в нову життєву ситуацію.
- Зазвичай одна найкраща відповідь у кожному кейсі.
- Ключі: `count`, `cases[{emoji, text, options[{label, correct, feedback?}]}]`.

### `quiz`
- Підсумкова перевірка.
- `type: "single"` або `type: "multiple"`.
- Підтримується пул питань + випадковий відбір `count`.

## 5) Чекліст педагогічної якості

- Є чітка мета уроку і очікувані результати.
- Є пояснення для учня + методичні підказки для вчителя (`teacherTip`, `goalNote`).
- Перший блок у режимі учня працює як зацікавлення, а не як службова картка уроку.
- Перший блок у режимі вчителя може містити тривалість, формат, організаційні особливості.
- Є формувальне оцінювання:
  - питання "чому?" після вибору
  - короткі мікроперевірки в підказках
- Є хоча б один елемент перенесення знання в нову ситуацію (`transfer`) або вчительська усна альтернатива до нього.
- Є хоча б один елемент самостійного конструювання відповіді (`creative`) або творча усна альтернатива.
- Є диференціація:
  - базовий рівень + опція для сильніших учнів
- Формулювання короткі, конкретні, віково зрозумілі (1-2 клас).
- У завданнях не вимагається друкування довгих відповідей.

## 6) Емодзі та сумісність

- Уникати нових/рідкісних емодзі, які часто дають "квадратик" на старих Windows.
- Віддавати перевагу стабільним: `📘 📱 🎵 🚦 🔔 ✅ ⚠️ ❗ ✍️ 🗿`.
- Якщо емодзі сумнівний — замінити на більш сумісний аналог.
- Не робити емодзі єдиним носієм змісту: текст має бути самодостатнім.

## 7) Технічні правила (щоб не плодити борг)

- Не додавати темо-специфічну логіку в `js/task-*.js` або `js/app.js`.
- Усе предметне — тільки в `js/lessons/*.js`.
- Не вмикати постійне збереження прогресу в `localStorage` або інших сховищах: це суперечить сценарію спільних шкільних пристроїв.
- Для нових типів активностей:
  1. створити `js/task-<type>.js`
  2. підключити в `js/activity-registry.js`
  3. додати гілку в `js/lesson-data.js` (побудова/рандомізація)

## 8) Definition of Done для нового уроку

- Є `js/lessons/<slug>.js`
- Є `<slug>.html`
- Є запис у `js/lessons/catalog.js`
- Урок відкривається з каталогу і напряму по URL
- Пройшов `tests/run-tests.ps1`
- Нема явних "квадратиків" замість емодзі
- Контент відповідає віку і має методичну цінність
- Після оновлення сторінки урок стартує з чистого стану

## 9) Мінімальний бриф від користувача (який треба попросити)

Щоб агент зробив урок "під ключ", достатньо:
1. Тема
2. Клас/вік
3. Ключові поняття (5-10 пунктів)
4. Бажані приклади з життя дітей
5. Обмеження (що не включати)
6. Мова і бажаний тон

Далі агент робить все сам: структуру, інтерактиви, тести, окремий URL, перевірки.

# Шаблон Нового Питання

Цей файл потрібен для безпечного додавання нового питання в `questions.js`.

## 1. Швидка перевірка перед додаванням

- Який `conceptKey` це питання міряє?
- Це `core` чи `extension`?
- Який рівень складності?
  - `foundational`
  - `developing`
  - `advanced`
- Яку мисленнєву дію воно вимагає?
  - `recognize`
  - `apply`
  - `reason`
- Чи додає воно нову сходинку концепту, а не дублює вже наявне?

## 2. Шаблон об'єкта питання

```js
{
    q: "СЮЖЕТНЕ ПИТАННЯ ДЛЯ ДИТИНИ",
    concept: "Короткий опис концепту",
    conceptKey: "obov-iazkovyi-kliuch",
    conceptLabel: "Людяна назва концепту",
    options: [
        "Правдоподібний, але хибний дистрактор 1",
        "Правильна відповідь",
        "Правдоподібний, але хибний дистрактор 2",
        "Правдоподібний, але хибний дистрактор 3"
    ],
    correct: 1,
    explanation: "Коротке пояснення, чому правильна відповідь справді найкраща."
}
```

Примітка:
- `id`, `skill`, `difficulty`, `progressionBand`, `isCore`, `reviewStatus`, `distractorQuality` зараз добудовуються автоматично в `questions.js`.
- Якщо ми колись переведемо ці поля на ручне заповнення, структура вже готова до цього.

## 3. Як обрати `difficulty`

- `foundational`
  Питання на впізнавання або пряме застосування в дуже прозорій побутовій ситуації.
- `developing`
  Питання вимагає трохи ширшого перенесення знання, перевірки плану або вибору між кількома схожими діями.
- `advanced`
  Питання містить складену умову, кілька критеріїв, багатокрокове міркування або потребує порівняння стратегій.

## 4. Як обрати `progressionBand`

- `recognize`
  Дитина має назвати, упізнати, продовжити очевидний патерн або знайти один правильний приклад.
- `apply`
  Дитина має застосувати концепт до нової, але ще прозорої ситуації.
- `reason`
  Дитина має пояснити, передбачити наступний крок, зіставити критерії або зробити висновок із кількох умов.

## 5. Правило співіснування з банком

Нове питання варто додавати тільки якщо виконується хоча б одна умова:

- воно закриває прогалину в `conceptKey` для певного класу;
- воно додає новий `progressionBand` для вже наявного концепту;
- воно підсилює дистрактори там, де існуюче питання занадто просте;
- воно є якісним `extension`, який не витісняє `core`.

Якщо питання не додає жодної нової цінності, краще не вставляти його в основний банк.

## 6. Коротка пам'ятка по класах

- 2 клас:
  Переважно `foundational` + `recognize`
- 3 клас:
  Переважно `developing`, часто `apply`, іноді місток із `foundational`
- 4 клас:
  Часто `advanced`, переважно `apply` або `reason`

## 7. Міні-чеклист дистракторів

- Чи всі 3 неправильні варіанти звучать правдоподібно?
- Чи не є правильна відповідь єдиною “нормальною”?
- Чи не можна вгадати відповідь без обчислювального мислення?
- Чи не дублює питання інше питання тією самою логікою?

# AGENTS.md — інструкції для Claude Code та інших агентів

## 1. Контекст проєкту

Дитяча інтерактивна книга "Привіт, ШІ!" українською мовою. Статичний сайт без бекенду. Аудиторія — учні початкової школи.

Пріоритети: **безпека дітей → доступність → простота → стабільність**.

## 3. Категорично заборонено

- Повертати Tailwind CDN або будь-який CSS-фреймворк через CDN
- Повертати Plausible або інші трекери
- Повертати HTML-рядки в контент `bookSections.js` (`content: '<p>...'`)
- Повертати стару модель де контент рендерився через `innerHTML`
- Блокувати zoom (`user-scalable=no`, `maximum-scale`)
- Просити дитину вводити персональні дані
- Зберігати персональні дані в `localStorage`
- Видаляти або ослаблювати accessibility-атрибути
- Прибирати focus outline без рівноцінної заміни
- Залишати невалідні `.js` файли
- Підключати npm/node_modules — проєкт має залишатися без білд-тулів

## 4. Правила роботи з контентом

### Додавання нового повідомлення

```js
// Активність
{ type: 'activity', activityType: 'classifier', content: { title, instruction, options, cards, completionMessage } }
// activityType: 'classifier' | '20questions'

// Репліка Розумка
{ type: 'ai', content: 'Текст відповіді.' }

// Репліка Розумка з блоками
{ type: 'ai', blocks: [
  { type: 'paragraph', text: 'Перший абзац.' },
  { type: 'list', items: ['Пункт 1', 'Пункт 2'] }
]}

// Репліка Тимка
{ type: 'user', content: 'Питання Тимка.' }

// Проза-розповідь (narrator)
{ type: 'narrator', blocks: [
  { type: 'paragraph', text: 'У класі було гучно.' },
  { type: 'quote', lines: ['— Привіт! — сказала вчителька.'] }
]}

// Зображення — завжди з alt
{ type: 'ai', content: '...', image: '005.webp', alt: 'Опис зображення для screen reader' }
```

### Заборонено в контенті

```js
// ❌ Ніколи так
{ type: 'ai', content: '<p class="bold">Текст</p>' }
{ type: 'ai', content: '<ul><li>Пункт</li></ul>' }
```

### Якщо потрібен новий тип блоку

1. Додати `case` у `renderBlock` в `renderers.js`
2. Додати стиль у `style.css`
3. Оновити схему-коментар у `bookSections.js`
4. Оновити `validator.js` якщо потрібна валідація

## 5. Правила роботи зі стилями

- Усі стилі — у `style.css`, BEM-подібні класи
- Не додавати `style=""` атрибути в JS-рендерерах без крайньої потреби
- Не підключати зовнішні CSS-файли без погодження
- Responsive через `@media (min-width: 768px)` і `@media (min-width: 1024px)`

## 6. Кодування українського тексту — критична увага

**Проблема:** Edit tool може конвертувати ASCII-апостроф `'` (U+0027) у типографські лапки `'`/`'` (U+2018/U+2019) всередині JS-рядків, що спричиняє `SyntaxError`.

**Правило:** для будь-яких змін у `bookSections.js` з українським текстом — використовувати `Write` з template literals або окремий `_patch.mjs` скрипт. Після виконання скрипту — видаляти його.

## 7. Сюжетна логіка

Кожен розділ = один крок підготовки Тимка до Тижня ШІ. Структура:

1. Завдання (вступна сцена у класі)
2–10. Діалоги Тимка з Розумком + артефакт (визначення, плакат, гра тощо)
11. Словничок
12. Для батьків та вчителів
13. Про автора

## 8. Перевірка після кожної зміни

```bash
node --check main.js renderers.js quiz.js navigation.js menu.js storage.js validator.js bookSections.js
```

Ручний тест:
- обкладинка → перший розділ → назад
- відкрити меню, перейти в довільний розділ
- пройти квіз
- перевірити консоль на помилки
- перевірити мобільну ширину

## 9. Критерії завершення задачі

- Код запускається без помилок у консолі
- `node --check` проходить для всіх JS
- Виконано конкретні acceptance criteria задачі
- Не додано нових privacy-ризиків
- Не зламано клавіатурну навігацію
- Контент залишився зрозумілим для дітей

## 10. Неочевидні рішення, які треба зберегти

- `messageTimers` очищається при кожному `loadSection` — не видаляти
- Квізи не карають за помилку — не змінювати логіку
- `#a11y-announcer` — окремий aria-live регіон, не chatWindow
- `chat-window--cover` і `bottom-nav--hidden` — пара для обкладинки
- Анімація переходу: exit (180ms) → enter (280ms) через `_renderSection`

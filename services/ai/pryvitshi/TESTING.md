# TESTING.md — план тестування “Привіт, ШІ!”

## 1. Мета тестування

Переконатися, що інтерактивна книга:

- відкривається без помилок;
- коректно показує всі розділи;
- працює на мобільних і десктопах;
- доступна з клавіатури;
- не містить privacy-ризиків;
- працює offline після впровадження PWA;
- має педагогічно коректний контент.

## 2. Мінімальні автоматичні перевірки

### 2.1. Синтаксис JS

```bash
node --check main.js
node --check bookSections.js
```

Поточна проблема:

```bash
node --check quizzes_updated.js
```

повертає синтаксичну помилку, бо файл не є самостійним JS-модулем. Його треба або інтегрувати, або перейменувати на Markdown/чернетку.

### 2.2. Пошук ризикових патернів

```bash
grep -R "user-scalable=no\|maximum-scale\|gtag\|plausible\|target=\"_blank\"\|innerHTML\|localStorage" .
```

Ця команда має показувати тільки очікувані й виправдані місця. Для release-версії не має бути:

- `user-scalable=no`;
- Google Analytics;
- Plausible;
- `target="_blank"` без `rel`;
- `innerHTML` для контентних даних.

## 3. Ручний smoke test

Перевірити після кожної значної зміни:

- [ ] Відкривається обкладинка.
- [ ] Кнопка старту переходить до вступу.
- [ ] Кнопка “Далі” працює.
- [ ] Кнопка “Назад” працює.
- [ ] Індикатор сторінки правильний.
- [ ] Sidebar показує всі розділи.
- [ ] Активний розділ підсвічується.
- [ ] Усі зображення завантажуються.
- [ ] Аудіо відтворюється.
- [ ] Квізи показуються.
- [ ] Правильна відповідь блокує варіанти.
- [ ] Неправильна відповідь дозволяє спробувати ще раз.
- [ ] У консолі немає помилок.

## 4. Тестування мобільного меню

На ширині 320–480px:

- [ ] Кнопка меню видима.
- [ ] Меню відкривається.
- [ ] Overlay з’являється.
- [ ] Кнопка закриття працює.
- [ ] Клік по overlay закриває меню.
- [ ] Escape закриває меню.
- [ ] Після закриття фокус повертається на кнопку меню.
- [ ] Немає горизонтальної прокрутки.

## 5. Accessibility-тести

### 5.1. Keyboard-only

Без миші:

- [ ] Tab переходить до skip-link.
- [ ] Enter на skip-link переводить до основного контенту.
- [ ] Tab проходить по меню, кнопках навігації, квізах.
- [ ] Enter/Space активує кнопки.
- [ ] Видно фокус на всіх інтерактивних елементах.
- [ ] Немає “пастки фокуса”, крім відкритого меню, якщо focus trap реалізований.

### 5.2. Zoom

- [ ] Сторінка читається при zoom 150%.
- [ ] Сторінка читається при zoom 200%.
- [ ] Немає горизонтального скролу.
- [ ] Кнопки не перекривають текст.

### 5.3. Reduced motion

Увімкнути reduced motion в OS/browser:

- [ ] Анімації прибрані або мінімізовані.
- [ ] Повідомлення не з’являються надто повільно.
- [ ] Користувач може читати без візуального “стрибання”.

### 5.4. Screen reader sanity check

Перевірити NVDA/VoiceOver або хоча б Chrome Accessibility Tree:

- [ ] Кнопки мають зрозумілі назви.
- [ ] Меню має коректний стан.
- [ ] Зображення мають змістовні alt.
- [ ] Квіз читається логічно.

## 6. Content QA

Для кожного розділу перевірити:

- [ ] Є життєва ситуація Тимка.
- [ ] Є зв’язок із Тижнем ШІ.
- [ ] Є один головний навчальний висновок.
- [ ] Розумко не перебільшує можливості ШІ.
- [ ] Є чесна згадка про помилки/перевірку інформації, де доречно.
- [ ] Немає складних термінів без пояснення.
- [ ] Немає фрази “ШІ все знає”.
- [ ] Немає поради вводити особисті дані.
- [ ] Квіз перевіряє саме зміст розділу.

## 7. Орфографія та стиль

Перевірити окремо:

- [ ] `обличчя`, не `обличча`.
- [ ] `малюнки`, не `молюнки`.
- [ ] `Сам ШІ`, не `Самий ШІ`.
- [ ] Український апостроф: `комп’ютерний`, `зв’язок`, `об’єкт`.
- [ ] Єдине написання `ШІ`.
- [ ] Речення не надто довгі для дітей.
- [ ] Діалоги звучать природно.

## 8. Security/privacy tests

- [ ] У HTML немає Google Analytics.
- [ ] У HTML немає Plausible.
- [ ] Немає інших сторонніх трекерів.
- [ ] Немає форм збору персональних даних.
- [ ] Немає запиту фото/адреси/телефону дитини.
- [ ] Зовнішні посилання мають `rel="noopener noreferrer"`.
- [ ] Новий контент не вставляється через небезпечний HTML.

## 9. PWA/offline tests

Після додавання PWA:

- [ ] Service worker зареєстрований.
- [ ] Cache Storage містить основні файли.
- [ ] Книга відкривається без мережі після першого завантаження.
- [ ] Зображення доступні offline.
- [ ] Аудіо доступне offline або має зрозумілий fallback.
- [ ] Після зміни `CACHE_NAME` старий кеш видаляється.

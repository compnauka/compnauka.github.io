# SECURITY.md — безпека та приватність “Привіт, ШІ!”

## 1. Модель продукту

`Привіт, ШІ!` — дитячий освітній статичний сайт без реєстрації. Основна аудиторія — учні початкової школи, вчителі та батьки.

Головний принцип:

> Сайт не має збирати, зберігати або передавати персональні дані дітей, якщо це не є абсолютно необхідним.

У поточній версії серверної частини немає, тому основні ризики знаходяться на рівні фронтенду, сторонніх скриптів, контенту, локального сховища та accessibility/privacy-рішень.

## 2. Поточні ризики

### 2.1. Стороння аналітика

У `index.html` підключено Google Analytics і Plausible. Для дитячого продукту без явної згоди це небажано.

**Рішення:** прибрати обидві аналітики за замовчуванням.

Якщо статистика буде потрібна пізніше, дозволені варіанти:

1. агреговані server logs без cookies;
2. self-hosted аналітика без ідентифікації користувача;
3. аналітика тільки після явної згоди дорослого.

### 2.2. `innerHTML` для контенту

У `main.js` багато вставок через `innerHTML`, а в `bookSections.js` контент містить HTML-рядки. Зараз контент локальний, але це створює майбутній XSS-ризик.

**Рішення:**

- новий контент зберігати у структурованому форматі;
- текст вставляти через `textContent`;
- медіа створювати через `createElement`;
- посилання створювати через DOM API з перевіркою URL;
- довільний HTML у контенті заборонити.

### 2.3. Зовнішні посилання

У `bookSections.js` є посилання з `target="_blank"` без `rel="noopener noreferrer"`.

**Рішення:** усі зовнішні посилання мають мати:

```html
rel="noopener noreferrer"
```

### 2.4. Зовнішні залежності

У `index.html` підключені Tailwind CDN, Google Fonts, Google Analytics, Plausible.

Ризики:

- залежність від доступності сторонніх сервісів;
- неможливість суворої CSP;
- слабший offline-режим;
- supply-chain ризики;
- додаткові privacy-запити.

**Рішення:**

- прибрати аналітику;
- перейти з Tailwind CDN на локально зібраний CSS або залишити Tailwind тільки на етапі прототипу;
- розглянути локальне підключення шрифту або системний fallback;
- після цього ввімкнути сувору CSP.

### 2.5. `localStorage`

Поточний прогрес зберігається в `localStorage` як `aiBookProgress`. Це не персональні дані, але на спільних шкільних комп’ютерах може збивати наступного учня.

**Рішення:**

- або перейти на `sessionStorage`;
- або показувати явний вибір “Продовжити / Почати спочатку”.

### 2.6. Блокування масштабування

У `index.html` заблоковано масштабування. Це accessibility-ризик і порушення стандартів.

**Рішення:**

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

## 3. Заборонені практики

У цьому проєкті заборонено:

- збирати ПІБ, телефон, адресу, email дитини;
- просити фото дитини;
- просити пароль або дані акаунта;
- підключати трекери без окремого рішення власника продукту;
- додавати рекламні скрипти;
- вставляти довільний HTML із контентних файлів;
- використовувати inline event handlers у контенті;
- зберігати персональні дані в `localStorage`;
- блокувати zoom;
- приховувати зовнішні переходи від користувача.

## 4. Дозволені дані

Без окремої згоди можна зберігати тільки технічний, неперсональний стан:

- номер поточного розділу;
- локальні налаштування читання;
- результат квізу в межах поточної сесії, якщо він не передається назовні.

Для дитячої аудиторії краще не зберігати історію відповідей довго.

## 5. Content Security Policy

Після відмови від CDN і аналітики цільова CSP:

```http
Content-Security-Policy: default-src 'self'; script-src 'self'; style-src 'self'; img-src 'self'; media-src 'self'; font-src 'self'; connect-src 'self'; object-src 'none'; base-uri 'none'; frame-ancestors 'none';
```

Якщо Tailwind CDN або Google Fonts тимчасово залишаються, CSP буде слабшою. Це допустимо тільки як перехідний стан.

## 6. Політика посилань

Для зовнішніх посилань:

```html
<a href="https://example.com" target="_blank" rel="noopener noreferrer">...</a>
```

Для дитячого продукту всі зовнішні ресурси мають бути вручну перевірені.

Поточні зовнішні ресурси в контенті:

- `https://hourofcode.com/ai-oceans`
- `https://quickdraw.withgoogle.com/`
- `https://itnauka.org/`

## 7. Безпечний рендеринг контенту

Рекомендований підхід:

```js
function renderParagraph(text) {
  const p = document.createElement("p");
  p.textContent = text;
  return p;
}
```

Небажаний підхід:

```js
contentDiv.innerHTML = message.content;
```

Якщо потрібно підтримати форматування, використовувати структуровані блоки:

```js
{ type: "strong", text: "важливо" }
{ type: "list", items: ["..."] }
{ type: "quote", text: "..." }
```

## 8. Безпека медіафайлів

- Усі зображення мають лежати локально.
- Усі аудіо мають лежати локально.
- Не підвантажувати зображення з невідомих URL.
- Не дозволяти користувацьке завантаження файлів у цій версії продукту.
- Для аудіо додавати опис або транскрипт.

## 9. Мінімальний security checklist перед публікацією

- [ ] У `index.html` немає Google Analytics.
- [ ] У `index.html` немає Plausible або інших трекерів.
- [ ] Немає `user-scalable=no`.
- [ ] Немає `maximum-scale=1.0`.
- [ ] Усі `target="_blank"` мають `rel="noopener noreferrer"`.
- [ ] Контент не вставляється через небезпечний `innerHTML`.
- [ ] Немає inline event handlers у контенті.
- [ ] Немає збору персональних даних.
- [ ] Немає форм, які просять дитину ввести особисті дані.
- [ ] Усі зовнішні посилання перевірені.
- [ ] У консолі браузера немає помилок.
- [ ] JS проходить синтаксичну перевірку.
- [ ] Сайт працює без інтернету після кешування PWA.

## 10. Позиція про аналітику

Рекомендація для цього проєкту: **не використовувати сторонню аналітику в дитячій книзі**.

Це зменшує юридичні ризики, підвищує довіру вчителів і батьків та спрощує архітектуру.

# Привіт, ШІ! — практична документація проєкту

## Призначення

`Привіт, ШІ!` — статична інтерактивна книга українською мовою для дітей про штучний інтелект. Поточна реалізація працює як діалог Тимка з Розумком: користувач переходить між розділами, читає повідомлення, переглядає ілюстрації, слухає аудіо та відповідає на короткі квізи.

Ціль проєкту — зробити безпечну, доступну, автономну й педагогічно цілісну інтерактивну книгу для використання в школі без реєстрації.

## Поточна структура файлів

```text
pryvitshi/
├── index.html              # HTML-каркас сторінки, підключення стилів і скриптів
├── style.css               # Додаткові стилі, анімації, responsive-правки
├── main.js                 # Логіка навігації, рендер повідомлень, квізи, прогрес
├── bookSections.js         # Контент книги: розділи, повідомлення, квізи, медіа
├── quizzes_updated.js      # Чернетка/невалідний JS, потребує інтеграції або перейменування
├── PROJECT_STANDARDS.md    # Загальні стандарти освітніх продуктів
├── DESIGN_AUDIT_REPORT.md  # Попередній аудит UX/доступності/безпеки
├── 000.webp ... 014.webp   # Ілюстрації
└── AI_SONG.mp3             # Аудіофрагмент для розділу про генеративний ШІ
```

## Поточний стан

Проєкт уже можна запускати як статичний сайт. Серверна частина, база даних і авторизація відсутні. Це хороша основа для дитячого продукту, бо зменшує кількість ризиків.

Водночас поточна реалізація має критичні розбіжності з власними стандартами проєкту:

1. У `index.html` заблоковано mobile zoom через `maximum-scale=1.0, user-scalable=no`.
2. Підключено сторонню аналітику Google Analytics і Plausible, що небажано для дитячого продукту без окремої згоди.
3. Немає PWA/offline-реалізації: відсутні `manifest.json`, `sw.js`, `offline.html`.
4. Контент і HTML-розмітка змішані в `bookSections.js`.
5. Значна частина рендерингу побудована через `innerHTML`.
6. Мобільне меню не має повного ARIA/focus-керування.
7. `quizzes_updated.js` є синтаксично невалідним JS-файлом.
8. Зображення мають загальні `alt`, а аудіо не має текстового супроводу.

## Як запустити локально

Найпростіше — відкрити `index.html` у браузері. Для коректного тестування ES-модулів краще запускати локальний статичний сервер:

```bash
python3 -m http.server 8080
```

Після цього відкрити:

```text
http://localhost:8080/
```

## Мінімальні технічні вимоги

- Сайт має працювати як статичний застосунок.
- Без логіну.
- Без збору персональних даних дітей.
- Без сторонніх трекерів за замовчуванням.
- З підтримкою клавіатури, touch і миші.
- З можливістю масштабування сторінки.
- З offline-режимом після впровадження PWA.

## Рекомендований порядок робіт

### Етап 1 — обов’язковий hotfix

1. Прибрати `maximum-scale=1.0, user-scalable=no` з `index.html`.
2. Прибрати Google Analytics і Plausible з `index.html`.
3. Додати `rel="noopener noreferrer"` до всіх зовнішніх посилань з `target="_blank"`.
4. Додати skip-link до основного контенту.
5. Додати `aria-expanded`, `aria-controls` і Escape-закриття для мобільного меню.
6. Додати `prefers-reduced-motion` у `style.css`.
7. Перейменувати `quizzes_updated.js` на `QUIZ_DRAFT.md` або інтегрувати його в основний контент.

### Етап 2 — архітектурний рефакторинг

1. Винести контент у структурований формат без довільного HTML.
2. Рендерити повідомлення через DOM API, а не через шаблонні HTML-рядки.
3. Розділити `main.js` на модулі: навігація, рендеринг, квізи, сховище, accessibility.
4. Додати валідацію структури контенту.
5. Перейти з `localStorage` на `sessionStorage` або додати явний вибір “Продовжити / Почати спочатку”.

### Етап 3 — PWA/offline

1. Додати `manifest.json`.
2. Додати `sw.js`.
3. Додати `offline.html`.
4. Закешувати основні файли, зображення й аудіо.
5. Перевірити роботу в режимі offline.

### Етап 4 — змістова перебудова під “Тиждень ШІ”

1. Переписати вступ як шкільну сцену оголошення Тижня ШІ.
2. Кожен розділ прив’язати до конкретної підготовчої дії Тимка.
3. Замінити розділ про машинне навчання на демонстрацію з картками.
4. Переробити розділ “Давай пограємо!” на активність для першачків.
5. Завершити книгу фінальною доповіддю Тимка.

## Критерії готовності першої стабільної версії

Версію можна вважати стабільною, якщо:

- немає сторонньої аналітики;
- mobile zoom не заблокований;
- сайт проходить ручний тест клавіатурної навігації;
- меню коректно відкривається й закривається на мобільному;
- усі зображення мають змістовний alt;
- аудіо має текстовий опис або транскрипт;
- контент не ламає верстку;
- сайт працює offline після першого відкриття;
- немає синтаксичних помилок у JS-файлах;
- сюжет має наскрізну логіку “Тимко готується до Тижня ШІ”.
